# AirMix 产品需求文档

## 产品一句话定义

AirMix 是一款手持式互动音乐设备，让用户在播放手机音乐时，通过动作和实体按钮实时改变音效，把被动听歌变成低门槛 DJ / 互动乐器体验。

## 用户场景

- 朋友聚会：用户用手机播放歌单，拿起 AirMix 晃动、倾斜、敲击，让副歌、drop 或节奏段出现滤波、门限、回声和切片效果。
- 独自听歌：用户戴耳机通过 3.5mm 输出听歌，用轻微动作控制滤波和节奏效果，像玩一个随身乐器。
- 短视频创作：用户边放音乐边做手势，把动作和音效同步录进视频，提高表现力。
- 音乐启蒙：不懂 DJ 软件或合成器的用户，也能通过自然动作理解 cutoff、gate、delay、stutter 等声音变化。

## 核心价值

- 低学习成本：动作即控制，不要求用户理解复杂音频术语。
- 强即时反馈：每次倾斜、晃动、敲击都能听到声音变化，形成乐器感。
- 便携独立：设备既可当蓝牙音箱，也可接耳机，不依赖电脑和专业控制器。
- 可扩展交互：同一套音频和动作架构未来可加入更多效果、灯光反馈、自动节拍检测和手势模型。

## MVP 功能

- 蓝牙音频接收端的抽象接口，第一阶段用模拟音频输入代替真实蓝牙。
- 音频输出抽象接口，第一阶段用模拟输出代替扬声器、DAC、功放和耳机口。
- IMU 抽象接口，第一阶段支持模拟加速度计和陀螺仪输入。
- 阈值式动作识别：
  - shake：快速晃动。
  - tilt left / tilt right：左右倾斜。
  - punch / hit：快速向前或向下冲击。
  - rotate：快速旋转。
  - lift / drop：上抬或下落。
- 实体按钮抽象：
  - Effect Mode：切换 High-pass / Low-pass / Gate / Delay / Stutter。
  - Hold：保持当前效果参数。
  - Tap：输入 tempo。
  - Bypass：关闭所有效果。
- buffer 级 DSP 原型：
  - High-pass filter。
  - Low-pass filter。
  - Gate / rhythmic mute。
  - Delay / echo。
  - Stutter / repeat。
  - Volume ducking。
- Tap Tempo 驱动节奏类效果。
- 控制台 demo 打印当前手势、效果模式和主要参数变化。

## 非 MVP 功能

- 真实蓝牙 A2DP 协议栈和认证级音频稳定性。
- 复杂自动 BPM 检测、beat tracking 或音乐结构识别。
- 机器学习手势识别。
- 多段动态压缩、limiter、studio-grade EQ 或高级混响。
- App 配网、固件升级、预设商店和社交分享。
- LED 灯光、屏幕 UI、触觉马达和电池管理的完整实现。
- 工业设计、声学腔体、EMC、跌落可靠性和量产测试治具。

## 关键技术风险

- 蓝牙音频延迟：A2DP 本身可能带来明显延迟，动作和音效反馈需要尽量在设备端低延迟完成。
- 音频 buffer 尺寸：buffer 太大导致手感迟滞，太小会增加 CPU 和中断压力。
- DSP 余量：多个效果叠加时需要避免削波、爆音和 CPU 峰值。
- 手势误触：真实手持场景中，走动、换手、放桌面都可能触发错误手势。
- 扬声器回授和机械震动：内置扬声器工作时会影响 IMU 数据，需要硬件隔振和软件滤波。
- 耳机输出安全：效果叠加可能造成瞬时音量过大，后续需要 limiter 和音量保护。
- 电源噪声：蓝牙、功放、IMU、DAC 共板时需要关注地噪和电源纹波。

## 后续版本路线

- V0.1 软件原型：完成当前仓库中的模拟输入、手势检测、效果映射和基础 DSP。
- V0.2 桌面音频验证：接入本机音频文件或实时音频流，验证声音效果和交互节奏。
- V0.3 开发板验证：在 ESP32 / Teensy / STM32 / Raspberry Pi 上接入真实 IMU 和音频 codec。
- V0.4 低延迟调优：优化 buffer、线程、中断、DMA 和 DSP 定点化策略。
- V0.5 声学与硬件样机：验证扬声器、耳机口、按键位置、握持姿态和外壳震动。
- V1.0 MVP 产品：稳定蓝牙输入、扬声器/耳机输出、tap tempo、核心手势和 5 个效果模式。
- V1.x 增强体验：加入灯光反馈、预设、App 配置、自动 onset detection 和更稳健的手势识别。
